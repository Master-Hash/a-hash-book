样式优化¶

我认为测试不应该是测试各种 syntax 样式是否正常工作1，这种事情应该交给 cheat sheet 做。（除非自己对编译器（这里是 sphinx）做了太多手脚）

真正的测试是，明确自己写作需要哪些独特的样式，然后加以优化。

代码块¶

暂时只支持 bash 和 python，其它语言很笨拙。

bash / 其它 shell¶

%%bash
for i in {1..10}
do
    echo "${i}" & # & 表示 fork 一个子进程，在后台运行（相当于运行某程序，然后 C-z）
done
wait
echo "END"
# 输出随机顺序的 1 - 10，然后 END

此处 %%bash 即使可以理解，但不是文档通用的办法。 更好的是专门的 File 块。

另一个方案是使用 shell kernel。

以上 Cell magic 无论是 %%bash 还是 %%sh，调用的 shell 都是默认 shell（本地是 /bin/bash）

但目前不能证明后者是否会调用 POSIX 模式下的 bash。
